# reprorepo: Assessing Reproducibility Checklist

This repository contains [issues](https://github.com/valentina-s/reprorepo/issues) with checklists to assess the reproducibility of a data science project. The checklists contains questions aiming to identify what can be added to the project and its documentation to make it more reproducible. While the initial goal is for team members to use the checklist to have a discussion on reproducibility, eventually they can convert checklist items into actual issues that can be refined and addressed.

The issues are organized into three categories: Data, Software, and Documentation:

The activity can be preceded by some tutorials/presentations on best practices for reproducible data science (e.g. [Reproducible Data Science](https://github.com/valentina-s/ReproducibleDataScience/blob/main/Repro-Tutorial-DSSG2024.pdf) ), so the audience can discuss which approaches and tools are relevant to their project. Alternatively, with support of facilitators experienced in these topics, participants can explore these topics on the fly through the resources in the checklist.

The checklist questions in a [document format](https://github.com/valentina-s/ReproducibleDataScience/blob/main/ReproducibilityChecklist.pdf) were used to discuss reproducibility of projects in the Data Science for Social Good program at University of Washington and were converted to github format for the OceanHackWeek'25.  The questionnaire is based on [“Assessing Reproducibility”](http://www.practicereproducibleresearch.org/core-chapters/2-assessment.html), by A. Rokem, B. Marwick, V. Staneva.
